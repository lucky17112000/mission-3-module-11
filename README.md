## Buffer and strem:

১. Buffer কী?

Buffer হলো মেমোরির একটা জায়গা যেখানে ডেটা পুরোপুরি একবারে রাখা হয়।

এটা Raw binary data (bytes) ধরে রাখে।

যখন আপনি ছোট ডেটা বা ফাইল পড়েন, তখন পুরোটা একসাথে Buffer এ চলে আসে।

সহজে ভাবো, একটা বাক্স যেখানে সব কিছু একসাথে রাখা হয়।

ফায়দা:

ব্যবহার সহজ।

ডেটা একসাথে পাওয়া যায়।

ক্ষতি:

বড় ফাইল হলে মেমোরি খরচ বেশি হয় কারণ সব ডেটা একসাথে লোড হয়।

২. Stream কী?

Stream হলো ডেটার ছোট ছোট টুকরো (chunk) পাঠানোর বা গ্রহণ করার একটা ধারাবাহিক প্রক্রিয়া।

এটা ডেটাকে একবারে পুরোটা না নিয়ে, টুকরো টুকরো করে পাঠায়।

বড় বড় ফাইল বা লাইভ ডেটা (যেমন ভিডিও, অডিও) হ্যান্ডেল করার জন্য Stream সবচেয়ে ভালো।

ভাবো, একটি পাইপের মতো যার মাধ্যমে ডেটা প্রবাহিত হচ্ছে।

ফায়দা:

বড় ডেটা বা ফাইলের জন্য খুবই কার্যকর।

মেমোরি খুব কম খরচ হয়।

ক্ষতি:

ব্যবহার একটু জটিল, ইভেন্ট হ্যান্ডলার দিয়ে ডেটা ম্যানেজ করতে হয়।

| বিষয়           | Buffer                   | Stream                                |
| -------------- | ------------------------ | ------------------------------------- |
| ডেটা ধরন       | পুরো ডেটা একবারে রাখা    | ডেটা টুকরো টুকরো করে পাঠানো           |
| মেমোরি ব্যবহার | বেশি                     | কম                                    |
| ব্যবহার        | ছোট ফাইল, সহজ কাজের জন্য | বড় ফাইল, ভিডিও, অডিও, লাইভ ডেটার জন্য |
| সহজতা          | সহজ                      | একটু জটিল                             |

Buffer আর Stream একসাথে কিভাবে কাজ করে?

Stream থেকে আসা প্রতিটি ডেটার chunk হলো আসলে একটি Buffer।

Stream হচ্ছে ধারাবাহিক ডেটার প্রবাহ, আর Buffer হচ্ছে ডেটার সেই ছোট ছোট অংশগুলো।
